# אתר קטלוגים — פריסה ב-Netlify או Netlify + Cloudflare R2

הפרויקט תומך עכשיו בשני מסלולים מסודרים:

1. **באנדל רגיל** — האתר והתמונות נטענים מאותה העלאה ל-Netlify.
2. **באנדל R2** — האתר עצמו עולה ל-Netlify, והתמונות נטענות מ-Cloudflare R2.

למסלול R2 נוסף סנכרון bucket אוטומטי: הוא עובר על `assets/pages`, מדלג על תמונות שכבר זהות ב-bucket, מעלה רק קבצים חדשים/שהשתנו, ומוחק מה-R2 קבצים שכבר לא קיימים מקומית. כך אין צורך להעלות ידנית קבוצות של 100 קבצים.

## מה מעלים ל-Netlify

מעלים את התיקייה:

```bat
dist\site-upload
```

התיקייה נוצרת על ידי:

```bat
bundle-site.bat
```

הבאנדל כולל רק את מה שהדפדפן צריך:

- קבצי האתר: `index.html`, `styles.css`, `app.js` ושאר קבצי JavaScript הנדרשים.
- נתוני הקטלוגים שנוצרו: `catalogs.generated.js`, `catalogs.search.js`.
- התמונות המומרות של הקטלוגים: `assets/pages/...`.
- `_headers` בשביל cache תקין ב-Netlify.

הבאנדל לא כולל קבצי עבודה שאינם נדרשים לצפייה באתר: PDF מקוריים, כלי המרה, README, virtualenv, קבצי הגדרות וסקריפטים פנימיים.


## לוח שליטה מקומי בדפדפן

במקום לזכור את כל קבצי ה־bat, אפשר לפתוח לוח שליטה מקומי:

```bat
catalog-control-panel.bat
```

הפקודה פותחת שרת מקומי ב־`127.0.0.1:8765` ודף בדפדפן. חשוב: לוח השליטה חייב להיפתח דרך `catalog-control-panel.bat`, כי רק השרת הזה מספק את כתובות ה־API כמו `/api/state`. אם פותחים את `catalog-control-panel.html` דרך `start-server.bat`, `npx serve` או השרת של האתר הראשי — הדף יעלה, אבל הכפתורים והקטלוגים לא יעבדו.

הדף מאפשר:

- להוסיף קבצי PDF חדשים ל־`catalogs.config.json`.
- לערוך ID, כותרת, תיאור שיופיע באתר, קטגוריה, תת־קטגוריה והגדרת OCR לכל קטלוג.
- למחוק קטלוג מהרשימה בלי למחוק את קובץ ה־PDF או את תיקיית התמונות.
- לשנות סדר בצורה היררכית: חצי קטגוריה מזיזים בלוק קטגוריה שלם, חצי תת־קטגוריה מזיזים בלוק תת־קטגוריה שלם, וחצי קטלוג מזיזים קטלוג בתוך אותה קטגוריה/תת־קטגוריה.
- להריץ המרה רגילה, המרה בכפייה, ניקוי קטלוגים לא רשומים, רענון OCR/חיפוש בלבד, סנכרון R2 ובאנדל.
- לראות יומן הרצה מלא בלי לפתוח חלונות bat נפרדים.

הדף לא נועד לעלות לאתר; הוא כלי עבודה מקומי בלבד. השרת מפעיל רק רשימת פעולות קבועה מראש, ולא מריץ פקודות חופשיות מהדפדפן. שמירת שינויים בלוח השליטה מקבצת קודם את הרשימה לפי קטגוריה ואז לפי תת־קטגוריה, תוך שמירת הסדר היחסי הקיים בתוך כל קבוצה. למשל, אם קטלוג שהיה רחוק ברשימה שונה לקטגוריה שכבר הופיעה בתחילת הקובץ, בשמירה הוא ייכנס לסוף אותה קטגוריה ולא יקפוץ לפי א״ב. לאחר מכן השמירה מעדכנת את `catalogs.config.json`, ובנוסף מסנכרנת כותרת/תיאור/קטגוריות/סדר/מחיקה אל `catalogs.generated.*` ו־`catalogs.search.*` עבור קטלוגים שכבר הומרו, כך שאין צורך לרנדר תמונות מחדש בשביל שינויי תוכן כאלה. אם משנים `id`, השמירה גם מחליפה את שם התיקייה המתאימה תחת `assets/pages` ומעדכנת את `catalogs.search-overrides.json` כדי שהגדרות החיפוש לפי עמודים יישארו מחוברות לקטלוג הנכון. קטלוג חדש שעדיין אין לו תמונות צריך עדיין הרצת המרה רגילה.

אפשר גם להריץ `start-server.bat`; הוא מציג תפריט בחירה בין האתר הראשי, לוח השליטה, או שניהם.

## סדר עבודה מומלץ

### 1. התקנה ראשונית במחשב

רק בפעם הראשונה:

```bat
setup-windows.bat
```

### 2. הנחת קבצי PDF מקוריים

שים את קבצי ה-PDF בתיקייה:

```bat
assets\pdfs
```

הנחת PDF בתיקייה עדיין לא ממירה אותו ולא משנה אוטומטית את האתר. כדי להוסיף קבצי PDF חדשים ל-`catalogs.config.json`, מריצים את סריקת ה-PDF הייעודית בשלב הבא.

### 3. סריקת PDF ועדכון `catalogs.config.json` בלבד

```bat
sync-catalog-pdfs.bat
```

הפקודה הזו רק סורקת את `assets\pdfs` ומוסיפה ל-`catalogs.config.json` קבצי PDF שלא רשומים עדיין בשום קטלוג. היא לא ממירה תמונות, לא מריצה OCR, ולא מעדכנת את `catalogs.generated.*`. אם שם PDF מכיל בטעות תווי כיוון נסתרים כמו `U+200F`, הסקריפט מנקה אותם משם הקובץ ומעדכן את הנתיב ב-JSON לנתיב נקי.

אחרי הסריקה ערוך ב-`catalogs.config.json` את `id`, `title`, `description`, `category`, `subcategory` ו-`ocr` לפי הצורך, או עשה זאת דרך `catalog-control-panel.bat`. `description` מוצג באתר מתחת לשם הקטלוג. `category` קובע את הכפתורים הראשיים בראש האתר. `subcategory` הוא אופציונלי: אם הוא ריק או חסר, הקטלוג נשאר מסודר כמו היום בתוך הקטגוריה הראשית; אם יש ערך, הקטלוג יוצג גם בתוך תת־קטגוריה עם כפתור פנימי לגלילה וסימון. סדר האובייקטים בקובץ קובע את סדר ההופעה באתר, כל עוד לא משתמשים בשדה `sort` ידני. דרך לוח השליטה השמירה מסדרת את הרשימה לבלוקים רצופים לפי `category` ואז `subcategory`; הסדר של הבלוקים נקבע לפי ההופעה הראשונה שלהם במצב הנוכחי, והסדר של הקטלוגים בתוך אותה תת־קטגוריה נשמר. שינוי `id` דרך לוח השליטה הוא שינוי מבוקר: אסור להשתמש ברווחים או בתווי נתיב, והשרת מחליף גם את שם `assets/pages/<id>` ומעדכן מפתח תואם ב-`catalogs.search-overrides.json`.

### 4. המרת הקטלוגים לתמונות

```bat
convert-catalogs.bat
```

הפקודה יוצרת/מעדכנת:

```bat
assets\pages
catalogs.generated.js
catalogs.generated.json
catalogs.search.js
catalogs.search.json
```

ברירת המחדל בפקודות הרגילות חזרה למה שהיה קודם: WebP מהיר לדפדוף, רינדור ב-220DPI, מגבלה של 2800px בצד הארוך, איכות 84 לעמודים, ותמונות ממוזערות בגודל 420px.

אם כבר קיימות תמונות תקינות ב-`assets/pages`, הסקריפט מדלג עליהן ולא ממיר מחדש בלי צורך. הבדיקה מתבססת על ה-PDF ועל הגדרות התמונה בלבד; שינוי בהגדרות OCR לא גורם לרינדור מחדש של תמונות. אם אינדקס החיפוש חסר/ריק, או שהגדרות ה-OCR השתנו, הסקריפט מרענן רק את טקסט החיפוש מתוך ה-PDF בלי לגעת בתמונות הקיימות. כדי לבנות מחדש את התמונות באיכות החדשה, הרץ `convert-catalogs-force.bat`.

אם יש קטלוג מסוים בלי כיתוב אמיתי, או קטלוג שבו OCR עלול לזהות שיבושים מתוך התמונה, הדרך המומלצת היא להגדיר לו ב-`catalogs.config.json`:

```json
"ocr": false
```

ברירת המחדל היא `true`, וגם קטלוג ישן בלי השדה הזה נחשב כאילו OCR פעיל. בהרצת `convert-catalogs.bat` רגילה, קטלוג עם `ocr: false` עדיין יומר לתמונות, אבל אינדקס החיפוש שלו ייבנה בלי OCR. אם התמונות כבר קיימות, הסקריפט ירענן את אינדקס החיפוש בלי לרנדר את התמונות מחדש, ככל שה-PDF המקורי עדיין קיים.

עדיין קיימת אפשרות גלובלית להמרה בלי OCR בכלל:

```bat
convert-catalogs-no-ocr.bat
```

הפקודה הזו שימושית לבדיקה או להרצה חד־פעמית בלי OCR לכל הקטלוגים. לעבודה שוטפת עדיף להשתמש בשדה `ocr` פר־קטלוג.

אם מחקת קטלוג מתוך לוח השליטה או ידנית מתוך `catalogs.config.json`, הוא מוסר מהרשימה ומהקבצים generated של האתר. אם אתה רוצה לנקות גם את תיקיית התמונות הישנה שלו מתוך `assets/pages`, הרץ `convert-catalogsdelete.bat`. הפקודה הזו עדיין מדלגת על קטלוגים קיימים שכבר הומרו, וממירה רק קטלוג חדש/חסר/שהשתנה.

### רענון OCR בלבד

אם רוצים לרענן רק את אינדקס החיפוש בלי לרנדר מחדש תמונות קיימות:

```bat
refresh-ocr-search.bat
```

ה-OCR במצב ברירת המחדל משתמש קודם בטקסט האמיתי שמוטמע ב-PDF, ומריץ OCR רק בעמודים סרוקים/ריקים מטקסט. לפני השליחה ל-Tesseract התמונה עוברת חיזוק ניגודיות וחידוד עדין כדי לשפר זיהוי של אותיות דקות/בהירות בעברית. הוא עדיין לא מנסה לנחש כותרות, סרטים צבעוניים או טקסט לבן מתוך חיתוכים אקראיים של התמונה עצמה, כדי לא להכניס תווים שגויים לאינדקס החיפוש.

### 5. סנכרון תמונות ל-Cloudflare R2 — רק במסלול R2

אם משתמשים במסלול שבו האתר ב-Netlify והתמונות ב-R2, מסנכרנים קודם את התמונות מתוך:

```bat
assets\pages
```

אל bucket ברירת המחדל:

```text
bargig-catalog
```

ובתוך ה-bucket הן נשמרות באותם נתיבים שהאתר מצפה להם:

```text
assets/pages/...
```

בפעם הראשונה צור קובץ פרטי בשם `r2.env` לפי הדוגמה:

```bat
copy r2.env.example r2.env
```

ומלא בו את:

```text
R2_ACCOUNT_ID=...
R2_ACCESS_KEY_ID=...
R2_SECRET_ACCESS_KEY=...
R2_PUBLIC_URL=https://cdn.bargig-furniture.com
```

הכתובת הציבורית של התמונות כבר מוגדרת כברירת מחדל:

```text
https://cdn.bargig-furniture.com
```

בדיקת תכנון לפני שינוי אמיתי:

```bat
sync-r2-images-preview.bat
```

הפקודה מציגה כמה קבצים יועלו, כמה יימחקו וכמה ידולגו כי הם כבר זהים. היא לא משנה כלום ב-bucket.

ביצוע הסנכרון בפועל:

```bat
sync-r2-images.bat
```

הסנכרון עושה שלושה דברים:

- מעלה תמונות חדשות.
- מעלה מחדש רק תמונות שהתוכן שלהן השתנה.
- מוחק מה-R2 קבצים שנמצאים תחת `assets/pages/` ב-bucket אבל כבר לא קיימים אצלך ב-`assets/pages`.

אם רוצים לעדכן בלי מחיקות, למשל לבדיקה זהירה:

```bat
sync-r2-images.bat --no-delete
```

אם רוצים לראות רשימה מלאה של כל הקבצים בתכנון ולא רק תקציר מקוצר:

```bat
sync-r2-images-preview.bat --show-all
```

> חשוב: `r2.env` מכיל מפתחות גישה ולא מעלים אותו ל-Netlify או ל-GitHub. הוא נוסף ל-`.gitignore`.

### 5. יצירת תיקיית העלאה נקייה ל-Netlify

ברירת המחדל נשארה כמו קודם: האתר והתמונות נבנים יחד, וכל `assets/pages` מועתק לתיקיית ההעלאה.

```bat
bundle-site.bat
```

תוצאה:

```bat
dist\site-upload
```

את התוכן של התיקייה הזו מעלים ל-Netlify.

אפשרות נוספת: באנדל אתר בלבד, בלי להעתיק את `assets/pages`, כשהתמונות נטענות מ-Cloudflare R2. לפני הבאנדל הזה מומלץ להריץ `sync-r2-images-preview.bat` ואז `sync-r2-images.bat`, כדי שה-bucket יהיה מעודכן מול התמונות הנוכחיות.

```bat
bundle-site-r2.bat
```

תוצאה:

```bat
dist\site-upload-r2
```

את התוכן של התיקייה הזו מעלים ל-Netlify. התמונות ייטענו מתוך:

```text
https://cdn.bargig-furniture.com/assets/pages/...
```

אם רוצים להשתמש בכתובת חיצונית אחרת במקום ה-R2 של ברגיג:

```bat
bundle-site.bat --external-assets-url https://cdn.example.com
```

אם רוצים גם ZIP של תיקיית ההעלאה:

```bat
bundle-site.bat --zip
```

זה ייצור בנוסף:

```bat
dist\site-upload.zip
```

## בדיקת תקינות מהירה

בבאנדל רגיל, בדוק שקיימים קבצים כמו:

```bat
dist\site-upload\index.html
dist\site-upload\assets\pages\<catalog-id>\page-001.webp
dist\site-upload\assets\pages\<catalog-id>\thumbs\page-001.webp
```

בבאנדל R2, בדוק שקיים קובץ ההגדרה ושהוא מכיל את כתובת R2:

```bat
dist\site-upload-r2\index.html
dist\site-upload-r2\catalog-assets.config.js
```

במצב R2 לא אמורה להיווצר בתוך תיקיית ההעלאה תיקיית `assets/pages`, כי התמונות כבר נמצאות באחסון החיצוני.

אם `bundle-site.bat` הרגיל נכשל עם הודעה ש-`assets/pages` לא קיים, המשמעות היא שלא בוצעה המרה עדיין או שהתמונות לא נמצאות בפרויקט. הרץ קודם `convert-catalogs.bat`, או השתמש ב-`bundle-site-r2.bat` אם התמונות כבר קיימות ב-R2.

## הוספת קטלוג חדש

1. העתק את ה-PDF לתוך `assets/pdfs`.
2. הרץ:

```bat
sync-catalog-pdfs.bat
```

3. אם ה-PDF עדיין לא היה רשום, הסקריפט יוסיף אותו ל-`catalogs.config.json` עם:
   - `id`: שם הקובץ בצורה בטוחה לשימוש כתיקייה
   - `title`: שם הקובץ בלי הסיומת `.pdf`
   - `description`: ריק
   - `category`: ריק
   - `pdf`: הנתיב לקובץ בתוך `assets/pdfs`
   - `ocr`: ברירת מחדל `true`; אפשר לשנות ל-`false` כדי למנוע OCR בקטלוג הזה גם בהרצה רגילה על כולם

   אם ה-PDF הגיע עם תווי כיוון נסתרים בשם הקובץ, למשל `U+200F`, הסקריפט ינקה אותם משם הקובץ לפני הכתיבה ל-JSON כדי שהנתיב יהיה קריא ורגיל.
4. ערוך ב-`catalogs.config.json` את הכותרת, התיאור, הקטגוריה והגדרת `ocr` לפי הצורך. אפשר לעשות זאת גם דרך `catalog-control-panel.bat`.
5. הרץ המרה:

```bat
convert-catalogs.bat
```

6. הרץ באנדל והעלה לאתר:

```bat
bundle-site.bat
```

7. העלה את `dist/site-upload` ל-Netlify.

```text
index.html                  דף האתר הראשי
styles.css                  עיצוב האתר
app.js                      לוגיקת הצופה והניווט
catalog-search.js           חיפוש בתוך הקטלוגים
catalog-snapshot.js         הורדת/צילום עמוד עם הלוגו
catalog-assets.config.js    הגדרת בסיס חיצוני לתמונות; ריק בבאנדל רגיל, כתובת R2 בבאנדל חיצוני
catalogs.config.json        רשימת הקטלוגים; קבצי PDF חדשים מתווספים אליו בהרצת sync-catalog-pdfs.bat; השדה ocr שולט על OCR פר קטלוג; שינוי id מומלץ לבצע דרך לוח השליטה
```

## מסלולי תמונות נתמכים

יש עכשיו שני מסלולים רשמיים:

1. באנדל רגיל: האתר טוען תמונות בנתיבים יחסיים רגילים, לדוגמה:

```text
assets/pages/qualita/page-001.webp
```

2. באנדל R2: אותם נתיבים יחסיים נשארים בנתוני הקטלוגים, אבל `catalog-assets.config.js` מוסיף להם בסיס חיצוני, לדוגמה:

```text
https://cdn.bargig-furniture.com/assets/pages/qualita/page-001.webp
```

הפתרון הזה עובד גם באתר הראשי וגם בדף `catalog-big-pages-viewer-netfree/catalog-big-pages-viewer.html`.
