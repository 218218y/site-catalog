# תכנית השקה עתידית לחיפוש Google

## מצב נוכחי

האתר נשאר בכוונה במצב פרטי מבחינת אינדוקס:

- לכל הדפים הציבוריים יש `meta robots` עם `noindex`.
- `_headers` מוסיף `X-Robots-Tag` חוסם.
- `robots.txt` מאפשר לסורקים לגשת לדפים כדי שיוכלו לקרוא את הוראת ה־`noindex`.

אין לפתוח את האתר לגוגל עדיין. האתר ממשיך להשתנות, וחשוב יותר לקבע קודם את
מבנה הכתובות, תוכן דפי הקטלוגים והמותג. שינוי כתובות או canonical לאחר תחילת
האינדוקס יוצר עבודת מעבר מיותרת.

## מה נכון להכין עכשיו בלי לפתוח אינדוקס

1. להשאיר את החסימה הקיימת ולשמור עליה בבדיקות הבנייה.
2. לבחור תמונת שיתוף קבועה ואיכותית למותג, ביחס המתאים להצגת קישורים.
3. להשלים תיאור יציב לכל קטלוג ב־`catalogs.config.json`.
4. להחליט אילו דפים ראויים להופיע בתוצאות חיפוש:
   - דף הבית;
   - דפי נחיתה סטטיים של קטלוגים;
   - תנאי שימוש ומדיניות פרטיות לפי הצורך.
5. להשאיר תמיד מחוץ לאינדוקס:
   - `favorites.html`;
   - קישורי שיתוף של מועדפים;
   - כתובות viewer עם מספר עמוד;
   - מסכי עזר וכלי ניהול מקומיים.

## המימוש המומלץ לפני ההשקה

### 1. מצב בנייה יחיד: private / public

להוסיף הגדרת build אחת שממנה נגזרים יחד:

- `meta robots`;
- `X-Robots-Tag`;
- `robots.txt`;
- יצירת `sitemap.xml`;
- בדיקות אוטומטיות שמונעות מצב מעורב.

ברירת המחדל צריכה להישאר `private`, והמעבר ל־`public` צריך להיות מפורש.

### 2. דפי נחיתה סטטיים לכל קטלוג

במקום לאנדקס כתובת כללית כמו `catalog.html?catalog=...`, מומלץ לייצר בזמן
הבנייה דף קבוע לכל קטלוג, לדוגמה:

```text
/catalogs/opening-tbi-2026/
```

כל דף יכלול מראש ב־HTML:

- כותרת ייחודית;
- תיאור ייחודי;
- canonical קבוע;
- תמונת כריכה;
- שם הקטגוריה;
- תקציר טקסטואלי;
- קישור לפתיחת הצופה.

כך מנוע החיפוש אינו תלוי בשינוי title/canonical בצד JavaScript.

### 3. Metadata ושיתוף

להוסיף בזמן הבנייה:

- Open Graph: `og:title`, `og:description`, `og:image`, `og:url`;
- תגי Twitter Card כתאימות כללית לשיתוף;
- תמונת שיתוף קבועה לדף הבית ותמונת כריכה לכל קטלוג;
- כותרות ותיאורים שאינם משוכפלים.

Open Graph מיועד בעיקר להצגת קישורים ברשתות וביישומי הודעות; הוא אינו תחליף
לכותרת, תיאור ו־structured data תקינים עבור Google.

### 4. מידע מובנה

בדף הבית להוסיף JSON-LD מסוג `LocalBusiness` או `Organization`, עם פרטים אמיתיים
בלבד:

- שם העסק;
- כתובת;
- טלפונים;
- שעות פתיחה בתיאום מראש;
- כתובת האתר;
- לוגו;
- מספר עוסק בשדה המתאים, אם נבחר סוג schema שתומך בכך.

אין להוסיף דירוגים, מחירים, מלאי או ביקורות שאינם מוצגים בפועל באתר.

### 5. Sitemap ו־canonical

- ליצור `sitemap.xml` מתוך הדפים הסטטיים שנבנו בפועל.
- לכלול רק כתובות canonical שרוצים באינדקס.
- להוסיף הפניה ל־sitemap בתוך `robots.txt` במצב public.
- canonical צריך להיקבע בשרת/ב־HTML שנבנה, לא להשתנות רק אחרי טעינת JavaScript.

### 6. בדיקות לפני פתיחה

לפני הסרת `noindex`:

1. להריץ `npm run verify`.
2. לבדוק את דפי הנחיתה ב־Rich Results Test.
3. לבדוק canonical, robots ו־sitemap על דומיין production.
4. לוודא שתמונות ה־CDN נגישות לסורקים.
5. לפתוח Google Search Console, לאמת בעלות ולשלוח sitemap.
6. לבקש אינדוקס למספר קטן של דפים ולבדוק את התוצאה לפני פתיחה רחבה.

## מתי לבצע

את מנגנון ה־private/public ודפי הנחיתה כדאי לבנות כאשר שמות הקטלוגים, הקטגוריות
ומבנה הכתובות יציבים. את הסרת החסימה עצמה עושים רק לאחר בדיקת production
מלאה. כרגע אין יתרון בפתיחה מוקדמת; נכון להמשיך לצבור נתוני שימוש ולסגור
שינויים במבנה האתר לפני שלב ההשקה.
