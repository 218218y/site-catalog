נכון לעכשיו האתר מוגדר להיות מוסתר מחיפוש גוגל

התקנה מקומית
setup-windows.bat

לוח שליטה נוח בדפדפן לכל פעולות הקטלוגים
catalog-control-panel.bat
חשוב: לא לפתוח את catalog-control-panel.html דרך start-server.bat או npx serve, כי שם אין API ולכן /api/state יחזיר 404.

אחרי הוספת קובץ pdf של קטלוג חדש לתקיית site-catalog\assets\pdfs
אפשר לעדכן את קובץ הקטלוגים אוטומטית עם
sync-catalog-pdfs.bat

אחר כך אפשר לערוך את הקטלוג החדש עם הפרטים כולל הקטגוריות והאם להריץ OCR בקובץ
catalogs.config.json

אפשר גם לערוך את זה דרך לוח השליטה: catalog-control-panel.bat
בלוח השליטה אפשר לערוך title / description / category / subcategory / OCR, למחוק קטלוג מהרשימה, ולשנות סדר עם כפתורי למעלה/למטה.
שמירה בלוח השליטה מעדכנת גם catalogs.generated.* עבור קטלוגים שכבר הומרו, לכן שינויי תיאור/שם/סדר/מחיקה נראים באתר בלי רינדור מחדש. קטלוג חדש עדיין צריך convert.

אחר כך להמיר את הpdf שלא הומרו
convert-catalogs.bat

אם רוצים גם למחוק תיקיות קטלוגים שכבר נמחקו מ-catalogs.config.json בלי להמיר מחדש קטלוגים קיימים
convert-catalogsdelete.bat

אחר כך יצירת באנדל לאתר אחד כולל אחסון תמונות
bundle-site.bat



אפשרויות נוספות


סדר עבודה מומלץ למסלול Netlify + Cloudflare R2
1. convert-catalogs.bat  - אם נוספו/השתנו PDF או תמונות מקומיות
2. sync-r2-images-preview.bat  - בדיקה יבשה מול bucket bargig-catalog
3. sync-r2-images.bat  - העלאת תמונות חדשות/שהשתנו ומחיקת ישנות
4. bundle-site-r2.bat  - יצירת תיקיית Netlify שמפנה לתמונות דרך https://cdn.bargig-furniture.com


בדיקת סנכרון תמונות ל-R2 בלי לבצע שינוי אמיתי
sync-r2-images-preview.bat

סנכרון תמונות ל-R2 בפועל: מעלה רק חדשות/שהשתנו ומוחק מה-bucket מה שכבר לא קיים מקומית
sync-r2-images.bat

יצירת באנדל חלוק בין אתר ראשי לאחסון תמונות במיקום אחר
bundle-site-r2.bat

המרת pdf חדשים/חסרים בלי OCR לקטלוגים שאין בהם כיתוב אמיתי
convert-catalogs-no-ocr.bat

המרה של כל pdf מחדש בהכרח(בלי למחוק קטלוג שכבר נמחק pdf שלו)
convert-catalogs-force.bat

המרת pdf חדשים בלבד + מחיקת קטלוגים שנמחקו מתוך catalogs.config.json
convert-catalogsdelete.bat

המרה של כל pdf מחדש בהכרח + מחיקת קטלוגים שנמחקו מתוך catalogs.config.json
convert-catalogs-deleteforce.bat

המרה מחדש רק של זיהוי ocr
refresh-ocr-search.bat

שרת מקומי לאתר הראשי בלבד
start-server.bat
או
npx serve .

לוח השליטה אינו עובד דרך שרת האתר הראשי. בשבילו מריצים catalog-control-panel.bat.
מחיקה דרך לוח השליטה מסירה רק מהרשימה ומהאתר שנוצר, לא מוחקת PDF ולא מוחקת assets/pages. לניקוי תיקיות ישנות מריצים convert-catalogsdelete.bat.

הגדרת OCR פר קטלוג
בקובץ catalogs.config.json אפשר להוסיף לכל קטלוג שדה:
"ocr": false
ואז גם אם מריצים המרה רגילה על כל הקטלוגים, הקטלוג הזה לא יריץ OCR. ברירת המחדל היא true.
